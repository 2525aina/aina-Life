# 111. ペット選択状態のページ間永続化

## 概要

この変更では、ユーザーが選択したペットの状態がページ遷移後も維持されるように、アプリケーションのグローバルな状態管理を導入しました。

## 変更前の問題点

前回の「ペット選択状態管理のリファクタリング」により、各ページコンポーネント（`page.tsx`, `history/page.tsx`, `settings/tasks/page.tsx`, `pets/page.tsx`）で選択中のペットの状態を一元管理するようになりました。しかし、ページを移動すると各ページコンポーネントのローカルな状態がリセットされるため、選択中のペットが常にリストの先頭に戻ってしまうという問題が発生しました。

## 変更内容

この問題を解決するため、ReactのContext APIを利用して、選択中のペットの状態をアプリケーション全体で共有・永続化する仕組みを導入しました。

### 1. `PetSelectionContext` の導入

-   `src/contexts/PetSelectionContext.tsx` を新規作成しました。
-   このファイル内で `PetSelectionContext` を定義し、`selectedPet` (選択中のペットオブジェクト) と `setSelectedPet` (選択中のペットを更新する関数) を提供します。
-   `PetSelectionProvider` コンポーネントを作成し、`usePets` フックを使用してペットのリスト (`pets`) とそのローディング状態 (`loading`) を取得し、`selectedPet` の状態を管理します。
-   `usePetSelection` というカスタムフックを提供し、各コンポーネントがContextの値を簡単に利用できるようにしました。

### 2. アプリケーションルートでの `Provider` の設定

-   `src/app/layout.tsx` を修正し、アプリケーション全体を `PetSelectionProvider` でラップしました。これにより、アプリケーション内のどのコンポーネントからでも `PetSelectionContext` の値にアクセスできるようになりました。

### 3. 各ページコンポーネントの修正

-   `Header` コンポーネントを利用するすべてのページ（`src/app/page.tsx`, `src/app/history/page.tsx`, `src/app/settings/tasks/page.tsx`, `src/app/pets/page.tsx`）を修正しました。
-   各ページからローカルな `selectedPet` の `useState` と `useEffect` を削除しました。
-   代わりに、`usePetSelection` カスタムフックを使用して、グローバルな `pets`, `loading`, `selectedPet`, `setSelectedPet` の値を取得するように変更しました。
-   これにより、ページ遷移後も選択中のペットの状態が維持されるようになりました。

### 4. `useLogbook` の利用箇所の調整

-   `useLogbook` フックは `petId` を引数として受け取るように変更済みであったため、各ページで `useLogbook(selectedPet?.id)` のように、Contextから取得した `selectedPet.id` を渡すように調整しました。

## 変更によるメリット

-   **状態の永続化**: ページ遷移後もユーザーが選択したペットの状態が維持されるようになり、ユーザー体験が向上しました。
-   **グローバルな状態管理**: 選択中のペットという重要な状態がアプリケーション全体で一元的に管理されるようになり、データフローがより明確になりました。
-   **コンポーネントの簡素化**: 各ページコンポーネントからローカルな状態管理ロジックが削除され、よりシンプルで読みやすいコードになりました。
