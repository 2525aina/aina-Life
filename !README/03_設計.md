# aina-Life(犬ライフログアプリ) 設計書

## 1. 概要
要件定義をもとに、React + Firebaseでの実装方法を具体化した設計。  
対象は最小追加要件まで。

---

## 2. データ構造（Firestore）

### 2.1 users コレクション

```json
users/{userId} {
authName: string,
authEmail: string,
nickname: string,
createdAt: Timestamp,
updatedAt: Timestamp
}
```

### 2.2 dogs コレクション
※ `ownerIds` は共有機能実装までの暫定的なフィールド。将来的には `members` サブコレクションに完全に移行する。
```json
dogs/{dogId} {
  name: string,
  breed: string,
  birthday: string,
  ownerIds: array<string>,
  createdBy: userId,
  updatedBy: Timestamp
  createdAt: Timestamp,
  updatedAt: Timestamp
}
```

### 2.3 members サブコレクション
（※ 家族と共有機能で実装）
```json
dogs/{dogId}/members/{userId} {
  role: "owner" | "general" | "viewer",
  invitedBy: userId,
  invitedAt: Timestamp,
  status: "pending" | "active" | "removed",
  inviteEmail: string, // 招待時のみ保持
  createdAt: Timestamp,
  updatedAt: Timestamp
}
```

### 2.4 tasks サブコレクション

```json
dogs/{dogId}/tasks/{taskId} {
  name: string,
  color: string,
  textColor: string, // (例: "#FFFFFF")
  order: number,
  createdBy: userId,
  updatedBy: Timestamp
  createdAt: Timestamp,
  updatedAt: Timestamp
}
```

### 2.5 logs サブコレクション

```json
dogs/{dogId}/logs/{logId} {
  petId: string,
  taskName: string,
  taskId: string,
  timestamp: Timestamp,
  note: string,
  inputType: "auto" | "manual",
  createdBy: userId,
  updatedBy: Timestamp
  createdAt: Timestamp,
  updatedAt: Timestamp
}
```

## 3. 画面構成

| 画面 | URL | 内容 |
|------|------|------|
| ログイン | /login | Googleログイン |
| ホーム | / | タスクボタン一覧、押下でログ記録、手入力フォーム |
| 履歴 | /history | 日付切替、ログ一覧、編集・削除 |
| タスク | /tasks | タスク追加・編集・削除、順序変更 |
| ペット | /pets | ペット追加・編集・削除、切り替え |
| 設定 | /settings | ユーザー情報、共有管理、ログアウト |


### 3.1 ホーム画面 設計詳細

**URLパス:** `/`

**目的:** 選択中のペットの行動を、ワンタップまたは簡単な手入力で素早く記録し、当日のログを時系列で確認する。

---

#### **レイアウト・コンポーネント**

*   **ヘッダー (`<Header />`)**
    *   **左側:** 現在選択中のペットの名前とアイコン画像を表示 (`<CurrentPetDisplay />`)
        *   タップするとペット選択画面（またはモーダル）に遷移。
    *   **右側:** 設定画面へ遷移する歯車アイコン (`<SettingsIcon />`)

*   **メインコンテンツ (`<Main>`)**
    *   **1. タスクボタンエリア (`<TaskSelector />`)**
        *   **役割:** 登録済みのタスクをボタンとして横一列に表示する。
        *   **UI:** 横スクロール可能なコンテナに、色分けされたタスクボタン (`<TaskButton />`) を配置。
        *   **操作:** ボタンをタップすると、現在時刻でそのタスクのログが記録される。
            *   例：「散歩」ボタンをタップ → `logs`コレクションに `{ taskName: "散歩", timestamp: (現在時刻) }` が追加される。

    *   **2. 今日のログタイムライン (`<LogTimeline />`)**
        *   **役割:** 本日のログを時系列（降順：新しいものが上）で表示する。
        *   **UI:** 各ログはカード形式 (`<LogItem />`) で表示。
            *   **表示内容:**
                *   時刻 (例: `14:32`)
                *   タスク名と色アイコン (例: 散歩)
                *   メモ（あれば表示）
                *   編集/削除ボタン
        *   **データ:** `dogs/{dogId}/logs` から `today` の `timestamp` を持つドキュメントをリッスンしてリアルタイムに更新。

    *   **3. フローティングアクションボタン (FAB) (`<QuickAddButton />`)**
        *   **役割:** 手動入力フォームをモーダルで表示するためのトリガー。
        *   **UI:** 画面右下に「＋」アイコンボタンを配置。
        *   **操作:** タップすると手動入力モーダルが開く。

*   **手動入力モーダル (`<ManualAddLogModal />`)**
    *   **役割:** 過去の時刻のログや、メモ付きのログを記録する。
    *   **UI:**
        *   時刻選択ピッカー (`<TimePicker />`)
        *   タスク選択ドロップダウン (`<TaskDropdown />`)
        *   メモ入力欄 (`<TextArea />`)
        *   保存ボタン

*   **フッターナビゲーション (`<FooterNav />`)**
    *   **役割:** 主要画面へのショートカット。
    *   **UI:** アイコン付きのナビゲーションリンクを配置。
        *   [ホーム] | [履歴] | [タスク] | [ペット]

### 3.2 履歴画面 設計詳細

**URLパス:** `/history`

**目的:** 日付を切り替えながら、過去のログを一覧で確認・管理（編集/削除）する。

---

#### **レイアウト・コンポーネント**

*   **ヘッダー (`<Header />`)**
    *   ホーム画面と共通のコンポーネントを再利用。

*   **メインコンテンツ (`<Main>`)**
    *   **1. 日付スイッチャー (`<DateSwitcher />`)**
        *   **役割:** 表示するログの日付を選択する。
        *   **UI:**
            *   「<」ボタン: 前の日に移動。
            *   現在選択中の日付 (例: `2025/08/25 (月)`)
                *   タップするとカレンダーピッカーが開き、任意の日付にジャンプできる。
            *   「>」ボタン: 次の日に移動。
            *   「今日」ボタン: ワンタップで今日の日付に戻る。
        *   **状態管理:** `selectedDate` の状態を管理し、変更に応じてFirestoreへのクエリを再実行する。

    *   **2. ログ一覧 (`<LogList />`)**
        *   **役割:** 選択された日付のログを時系列（降順）で表示する。
        *   **UI:** ホーム画面の `<LogTimeline />` と同様の `<LogItem />` コンポーネントを再利用する。
            *   **表示内容:** 時刻, タスク名, メモ, 編集/削除ボタン。
        *   **データ:** `dogs/{dogId}/logs` から `selectedDate` に合致する `timestamp` を持つドキュメントをクエリして表示する。
        *   **空の状態:** ログが1件もない日には、「この日の記録はありません」といったメッセージを表示する。

    *   **(オプション) その日のサマリー (`<DailySummary />`)**
        *   **役割:** 選択日のタスク回数を集計して表示する。
        *   **UI:** 「散歩: 2回」「ごはん: 3回」のように、タスクごとの回数をリスト表示する。

*   **フッターナビゲーション (`<FooterNav />`)**
    *   **役割:** 主要画面へのショートカット。
    *   **UI:** ホーム画面と共通のコンポーネントを再利用。
        *   [ホーム] | [履歴] | [タスク] | [ペット]

---

### 3.3 タスク画面 設計詳細

**URLパス:** `/tasks`

**目的:** ログ記録に使用するタスクの種類、色、表示順をユーザーが自由にカスタマイズできるようにする。

---

#### **レイアウト・コンポーネント**

*   **ヘッダー (`<Header />`)**
    *   **中央:** 画面タイトル「タスク」を表示。
    *   **左側:** 前の画面に戻る「<」アイコン。

*   **メインコンテンツ (`<Main>`)**
    *   **1. タスク一覧 (`<TaskList />`)**
        *   **役割:** 登録済みの全タスクをリスト形式で表示・管理する。
        *   **UI:**
            *   各タスクは一行のコンポーネント (`<TaskItem />`) として表示。
            *   ドラッグ＆ドロップで並べ替えられるように、各項目の左側にハンドル (`<DragHandle />`) を表示。
        *   **データ:** `dogs/{dogId}/tasks` から `order` フィールドでソートして全件取得する。

    *   **2. タスク項目 (`<TaskItem />`)**
        *   **役割:** 個々のタスク情報を表示し、編集・削除の起点となる。
        *   **UI:**
            *   タスクの色を示すカラーチップ (`<ColorChip />`)
            *   タスク名 (`<TaskName />`)
            *   編集ボタン (`<EditButton />`): タップすると編集モーダルが開く。
            *   削除ボタン (`<DeleteButton />`): タップすると確認モーダルが表示され、承認後に削除。
        *   **備考:** デフォルトタスクは削除不可とし、削除ボタンを非表示または無効化する。

    *   **3. 新規タスク追加ボタン (`<AddTaskButton />`)**
        *   **役割:** 新規タスク追加モーダルを開く。
        *   **UI:** 画面下部に固定表示するか、リストの末尾に配置する。「＋ 新しいタスクを追加」のようなボタン。

*   **タスク編集/追加モーダル (`<TaskEditModal />`)**
    *   **役割:** タスクの新規作成と既存タスクの編集を行う。
    *   **UI:**
        *   タスク名入力欄 (`<TextInput />`)
        *   カラーピッカー (`<ColorPicker />`)
        *   保存ボタン

---

### 3.4 ペット画面 設計詳細

**URLパス:** `/pets`

**目的:** ユーザーが所有するペットの情報を管理（追加・編集・削除）し、ログ記録の対象となるペットを切り替える。

---

#### **レイアウト・コンポーネント**

*   **ヘッダー (`<Header />`)**
    *   **中央:** 画面タイトル「ペット」を表示。
    *   **右側:** 新しいペットを追加するための「＋」アイコン (`<AddPetButton />`)。

*   **メインコンテンツ (`<Main>`)**
    *   **1. ペット切り替えタブ (`<PetSwitcherTabs />`)**
        *   **役割:** 登録されているペットを切り替えて表示する。
        *   **UI:** 各ペットの名前が入ったタブを横並びに表示。現在選択中のタブがハイライトされる。
        *   **データ:** `dogs` コレクションから `ownerIds` に自分のIDが含まれるドキュメントを全て取得し、タブを生成。

    *   **2. プロフィールカード (`<ProfileCard />`)**
        *   **役割:** 選択中ペットの詳細情報を表示する。
        *   **UI:** カード形式で以下の情報を表示。
            *   ペットのプロフィール写真 (将来拡張)
            *   名前 (`<PetName />`)
            *   犬種 (`<PetBreed />`)
            *   誕生日 (`<PetBirthday />`)
            *   年齢 (誕生日から自動計算して表示)

    *   **3. アクションボタンエリア (`<ActionButtons />`)**
        *   **役割:** プロフィール情報の編集や共有設定を行う。
        *   **UI:**
            *   「プロフィールを編集」ボタン: 編集モーダルを開く。
            *   「家族と共有」ボタン: 共有設定モーダルを開く。
            *   「このペットを削除」ボタン (危険な操作なので、目立たないスタイルにする): 確認モーダルを開く。

*   **プロフィール編集/追加モーダル (`<PetEditModal />`)**
    *   **役割:** ペットの新規登録と既存情報の編集を行う。
    *   **UI:**
        *   プロフィール写真アップロード欄
        *   名前入力欄 (`<TextInput />`)
        *   犬種入力欄 (`<TextInput />`)
        *   誕生日選択ピッカー (`<DatePicker />`)
        *   保存ボタン

*   **家族共有モーダル (`<FamilyShareModal />`)**
    *   **役割:** このペットの情報を共有する他のユーザーを管理する。
    *   **UI:**
        *   共有中のユーザー一覧 (`<SharedUserList />`)
        *   新しいユーザーをメールアドレスで招待する入力欄と追加ボタン。

*   **フッターナビゲーション (`<FooterNav />`)**
    *   **役割:** 主要画面へのショートカット。
    *   **UI:** ホーム画面と共通のコンポーネントを再利用。

## 5. 処理フロー

### 5.1 タスク記録
1. ボタン押下 → 現在時刻取得
2. Firestore logs に保存
3. UI更新（色分け）
4. 編集・削除可能

### 5.2 手入力
1. 時刻ピッカー + タスク名入力
2. Firestore logs に保存

### 5.3 履歴閲覧・編集
1. 日付切替
2. 当日の logs を取得・表示
3. 編集・削除・Undo可能
