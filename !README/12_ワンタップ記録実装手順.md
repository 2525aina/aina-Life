# aina-Life 実装手順書 - ワンタップ記録機能

## 1. 概要

このセクションでは、犬の行動をワンタップで記録する機能の実装手順を説明します。
ユーザーがタスクボタンをタップすると、現在時刻とタスク名が自動的にFirestoreの`logs`コレクションに記録されます。

## 3. 動作確認

実装した機能が正しく動作するか確認するために、Firestoreに手動でテストデータを作成します。

### 3.1. Firebaseコンソールでの設定とデータ登録

#### データベースの作成（初回のみ）
まだFirestoreデータベースを作成していない場合は、以下の手順を先に行います。

1.  **Firebase Console** にアクセスし、プロジェクトを選択します。
2.  左側メニューの「ビルド」から**「Firestore Database」**に移動します。
3.  **「データベースの作成」** ボタンをクリックします。
4.  **本番環境モードで開始** を選択し、「次へ」をクリックします。
5.  **Cloud Firestoreのロケーション** を選択します（例: `asia-northeast1` (東京)）。**この設定は後から変更できません。**
6.  「有効にする」をクリックします。

#### セキュリティルールの設定（開発初期）
データベース作成直後はデータが保護されているため、アプリからアクセスできません。開発中は、ログインユーザーに読み書きを許可するルールに変更します。

1.  Firestore画面の**「ルール」**タブを開きます。
2.  エディタの内容を以下に書き換えて**「公開」**します。

    ```
    rules_version = '2';
    service cloud.firestore {
      match /databases/{database}/documents {
        match /{document=**} {
          allow read, write: if request.auth != null;
        }
      }
    }
    ```
    **警告:** このルールは開発用です。本番リリース前には、より厳格なルールに設定する必要があります。

#### テストデータの登録

1.  **`dogs`コレクションの作成**: 「コレクションを開始」から `dogs` を作成します。
2.  **テスト用ペットの作成**: ドキュメントID `dog1` で、`name`フィールドを持つドキュメントを作成します。
3.  **`tasks`サブコレクションの作成**: `dog1`ドキュメントの中に `tasks` サブコレクションを作成します。
4.  **タスクデータの登録**: `tasks`コレクションに、`name`(string), `color`(string), `order`(number) のフィールドを持つドキュメントを複数作成します（例: ご飯, 散歩, 寝た）。

### 3.2. アプリケーションの起動と確認

1.  ターミナルで `npm run dev` を実行します。
2.  ブラウザでアプリを開き、Googleログインします。
3.  登録したタスクがボタンとして表示されることを確認します。
4.  ボタンをクリックし、Firebaseコンソールの `dogs/dog1/logs` に新しいログデータが記録されることを確認します。
