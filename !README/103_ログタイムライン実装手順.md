# aina-Life 実装手順書 - ログタイムライン機能

## 1. 概要

このセクションでは、記録された犬の行動ログをホーム画面にリアルタイムで表示する「ログタイムライン」機能の実装手順を計画します。
ユーザーがワンタップで記録した内容が、即座に画面に反映されることを目的とします。これは設計書のホーム画面における「今日のログタイムライン (`<LogTimeline />`)」に相当します。

---

## 2. 実装手順

### 2.1. `useLogbook`フックの機能拡張

`src/hooks/useLogbook.ts`に、今日のログを取得・管理する機能を追加します。

- **状態(State)の追加**: `logs`を保持するための新しいstateを追加します。
- **Firestoreリスナーの追加**: `useEffect`内に、`logs`サブコレクションを監視する`onSnapshot`リスナーを追記します。
- **返り値の追加**: フックが返すオブジェクトに`logs`を追加します。

**注意**: このフックが正しく動作するためには、Firestoreにペットデータ（`dogs/{dogId}`）と、そのペットに紐づくタスクデータ（`dogs/{dogId}/tasks`）が存在する必要があります。詳細は`12_ワンタップ記録実装手順.md`のテストデータの登録セクションを参照してください。

### 2.2. ログタイムラインUIコンポーネントの作成

取得したログデータを画面に表示するためのUIコンポーネント (`<LogTimeline />`) を`src/components`に作成します。

### 2.3. メインページへの統合

作成した`<LogTimeline />`コンポーネントをメインページ(`src/app/page.tsx`)に配置します。

### 2.4. 実装後のバグ修正 (`where`関数未定義エラー)

- **事象**: 上記実装後、`Runtime ReferenceError: where is not defined` という実行時エラーが発生。
- **原因**: `useLogbook.ts`内でFirestoreのクエリ関数`where`を使用しているにもかかわらず、`firebase/firestore`からインポートされていなかったため。
- **対応**: `useLogbook.ts`のインポート文に`where`を追加して修正。

## 3. 完了の定義

以下の要件が満たされた時点で、この機能の実装は完了とします。

- [ ] ホーム画面に、今日記録されたログが時系列（新しいものが上）で表示される。
- [ ] タスクボタンをクリックすると、ページをリロードすることなく、ログタイムラインに新しい記録が即座に追加される。
- [ ] 今日のログがまだ一件もない場合は、「今日の記録はまだありません」などのメッセージが表示される。
