# 112_家族と共有機能実装手順

## 0. 概要

このドキュメントは、犬ライフログアプリ「aina-Life」における「家族と共有機能」の実装要件とロードマップを整理したものです。

## 1. 開発環境のセットアップ (Firebase Local Emulator Suite)

「家族と共有機能」の実装とテストを安全かつ効率的に行うため、Firebase Local Emulator Suite を使用してローカル開発環境を構築します。これにより、本番環境のデータに影響を与えることなく、機能開発を進めることができます。

### 1.1. Firebase CLIの確認とインストール

Firebase CLIがインストールされていることを確認します。バージョンが古い場合はアップデートしてください。

```bash
firebase --version
# もしインストールされていない、またはバージョンが古い場合
npm install -g firebase-tools
```

### 1.2. Firebaseエミュレータの初期化

プロジェクトでFirebaseエミュレータを使用するための初期設定を行います。

```bash
firebase init emulators
```

質問には以下のように回答してください:

- `Which Firebase features do you want to set up for this directory?`: `Authentication Emulator` と `Firestore Emulator` を選択
- `Which port do you want to use for the auth emulator?`: `9099` (デフォルト)
- `Which port do you want to use for the firestore emulator?`: `8080` (デフォルト)
- `Would you like to enable the Emulator UI?`: `Yes`
- `Which port do you want to use for the Emulator UI?`: 空欄のままEnter (自動割り当て)
- `Would you like to download the emulators now?`: `Yes`

### 1.3. Firebaseエミュレータの起動

エミュレータを起動します。このコマンドは開発中は常に実行しておく必要があります。

```bash
firebase emulators:start
```

### 1.4. アプリケーションからのエミュレータ接続設定

アプリケーションがローカルエミュレータに接続するようにコードを修正します。

#### `src/lib/firebase.ts` の変更

`src/lib/firebase.ts` を開き、`getAuth` と `getFirestore` の呼び出しの後に、以下のエミュレータ接続ロジックを追加します。

```typescript
// src/lib/firebase.ts

import { initializeApp, getApps, getApp } from 'firebase/app';
import { getAuth, connectAuthEmulator } from 'firebase/auth';
import { getFirestore, connectFirestoreEmulator } from 'firebase/firestore';

// ... (firebaseConfig の定義はそのまま)

const app = !getApps().length ? initializeApp(firebaseConfig) : getApp();

const auth = getAuth(app);
const db = getFirestore(app);

// ローカルエミュレータを使用する場合の接続設定
if (process.env.NODE_ENV === 'development') {
  if (process.env.NEXT_PUBLIC_USE_FIREBASE_EMULATOR === 'true') {
    connectAuthEmulator(auth, 'http://localhost:9099');
    connectFirestoreEmulator(db, 'localhost', 8080);
  }
}

export { app, auth, db };
```

#### `.env.local` の変更

`aina-Life/.env.local` ファイルに以下の環境変数を追加します。

```
NEXT_PUBLIC_USE_FIREBASE_EMULATOR=true
```

### 1.5. 開発サーバーの再起動

環境変数の変更を反映させるため、Next.js開発サーバーを再起動してください。

```bash
# 停止
Ctrl+C
# 再起動
npm run dev
```

## 2. 機能概要

ユーザーが所有するペットの情報を、他のユーザーと共有し、共有されたペットに対してロールベースのアクセス制御を行う機能です。

---

### 2.1. ロール別権限比較表

| 機能/操作 | `owner` (所有者) | `general` (一般) | `viewer` (閲覧者) | 備考 |
| :--- | :---: | :---: | :---: | :--- |
| **ペット情報** | | | | |
| 閲覧 | ✅ | ✅ | ✅ | |
| 編集 | ✅ | ✅ | ❌ | |
| 削除 | ✅ | ❌ | ❌ | |
| **ログ** | | | | |
| 閲覧 | ✅ | ✅ | ✅ | |
| 追加 | ✅ | ✅ | ❌ | |
| 編集 | ✅ | ✅ | ❌ | |
| 削除 | ✅ | ✅ | ❌ | |
| **タスク** | | | | |
| 閲覧 | ✅ | ✅ | ✅ | |
| 追加 | ✅ | ✅ | ❌ | |
| 編集 | ✅ | ✅ | ❌ | |
| 削除 | ✅ | ❌ | ❌ | |
| **共有管理** | | | | |
| ユーザーの招待 | ✅ | ✅ | ✅ | ユーザー招待時はデフォルトで閲覧者 |
| 共有ユーザーのロール変更 | ✅ | 🔼 | ❌ | 一般は閲覧者のロールを一般にすることができる |
| 自身の共有解除 | ✅ | ✅ | ✅ | |
| 他人の共有解除 | ✅ | 🔼 | ❌ | 一般は閲覧者の共有を解除することができる |

## 3. データ構造（Firestore）

### 3.1 users コレクション

```json
users/{userId} {
  authName: string,
  authEmail: string,
  nickname: string,
  breed: string,
  birthday: string,
  createdAt: Timestamp,
  updatedAt: Timestamp
}
```

### 3.2 dogs コレクション

> 'ownerIds' は共有機能実装までの暫定的なフィールド。将来的には `members` サブコレクションに完全に移行する。

```json
dogs/{dogId} {
  name: string,
  breed: string,
  birthday: string,
  weight: string,
  ownerIds: array<string>,
  createdBy: userId,
  updatedBy: userId,
  createdAt: Timestamp,
  updatedAt: Timestamp
}
```

### 3.3 members サブコレクション

> 家族と共有機能で実装

```json
dogs/{dogId}/members/{userId} {
  role: "owner" | "general" | "viewer",
  invitedBy: userId,
  invitedAt: Timestamp,
  status: "pending" | "active" | "removed" | "declined",
  inviteEmail: string,
  createdAt: Timestamp,
  updatedAt: Timestamp
}
```

## 4. 実装機能詳細

### 4.1. ペット共有機能
- メール招待で共有可能
- 共有ユーザー一覧表示・管理
- 招待時はデフォルトで viewer 付与

### 4.2. ロールベースアクセス制御
- 3ロール（owner / general / viewer）による操作ロールベースアクセス制御
- 詳細は比較表参照

### 4.3. 設定画面統合
- /settings に専用画面を設置
- ユーザー情報表示（メール名・メールアドレス・ニックネーム・性別・誕生日）
- ペット共有管理セクション配置
- ログアウト機能

### 4.4. UI制御
- 各画面でロールに応じた操作可否を反映（表示/非表示、有効/無効）
