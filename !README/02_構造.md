# aina-Life(犬ライフログアプリ) アーキテクチャ
> Firebase 無料枠運用と技術スタック案（犬の行動管理アプリ）

2025年8月時点の情報に基づく、Web UI必須・将来的モバイル対応を前提とした整理です。

---

## 1. 無料枠での導入コスト比較

| 機能 | Firebase 無料枠条件 | 犬アプリでの見積もり | 注意点 |
|------|-------------------|--------------------|--------|
| Hosting | 月 1GB ストレージ、10GB 転送 | Web UI + 静的ページなら十分 | SSR/Next.js ISR は無料枠の転送量注意 |
| Data Connect | 無料枠なし（従量課金のみ） | 無料枠のみ運用は不可 | 代替は Firestore（無料枠有） |
| Firestore | 50k 書き込み/月、50k 読み込み/月、1GB ストレージ | 分単位ログ＋複数犬でも小規模なら無料枠内 | データ構造設計重要、統計処理はクエリ課金 |
| Realtime Database | 1GB ストレージ、10GB 転送/月 | 分単位ログ向きだが NoSQL 特有の集計制限 | 大規模分析は Cloud Functions 必須 |
| Crashlytics | 無料 | 制限なし | 問題なし |
| Remote Config | 無料 | 条件付き UI 切替に利用可能 | 100件/月制限あり |
| App Distribution | 無料 | テスター配布に利用可能 | iOS は TestFlight併用も検討 |
| AI Logic SDKs | 無料枠なし（従量課金） | 小規模なら外部無料AI APIを代用可能 | Firebase側で完全無料は不可 |

### 結論
- 完全無料運用の初期構成:
  - フロント：Next.js + Firebase Hosting
  - データベース：Firestore（無料枠内で小規模運用）
  - 認証：Firebase Authentication（無料枠あり）
  - クラッシュ検知：Crashlytics
  - UI設定切替：Remote Config（必要に応じ）
- Data ConnectやAI Logicは無料枠では不可 → 後から従量課金で導入可能

---

## 2. 技術スタック・データベース選び方

| 要件 | 推奨スタック・理由 | 無料枠対応 |
|------|-----------------|------------|
| Web UI 必須 | Next.js（React） | Hosting 無料枠で可能 |
| バックエンド | Firebase Functions（Node.js） | 無料枠 125,000 呼び出し/月、軽量ロジックなら十分 |
| データベース | Firestore | 1GB ストレージ、50k 書き込み/月、50k 読み込み/月で小規模運用可 |
| 将来的モバイル | React Native or Flutter + Firebase SDK | 認証・Firestore・Crashlytics 互換あり |
| 認証 | Firebase Authentication | メール/Google/GitHub など無料で利用可 |
| 分析・統計 | Firestore + Cloud Functions | Cloud Functions 無料枠で簡易集計可能（重い統計は課金注意） |

### 運用戦略
1. 最初は Web 専用  
   - Hosting + Firestore + Functions + Authentication  
   - 分単位ログを Firestore に保存  
2. 統計・分析機能  
   - Functions で日次集計 → Firestore に保存  
3. 将来的モバイル対応  
   - Web UI ほぼそのまま React Native へ移植可能  
   - Firebase SDK で iOS/Android 両対応  

---

## 3. 無料枠運用上の注意
- 分単位ログが多い場合 → Firestore 読み書き上限に注意
- 画像や動画保存は課金対象 → 犬の写真を大量保存すると無料枠超過
- AI Logic SDK は無料枠なし → 必要なら外部 API を使う

---

## 4. 推奨構成まとめ
- フロント：Next.js + Firebase Hosting  
- バックエンド：Firebase Functions (Node.js)  
- DB：Firestore  
- 認証：Firebase Authentication  
- 分析：Functions + Firestore  
- クラッシュ：Crashlytics  
- UI切替：Remote Config  
- モバイル拡張：React Native / Flutter（Firebase SDK利用）
