# 07_設計と実装の差分チェック

## 1. 概要
このドキュメントは、設計書（`03_設計.md`）で定義された要件と、現在のソースコード実装状況との間の差分を詳細にリストアップしたものです。

---

## 2. データ管理レイヤー (Hooks)

### `useAuth.ts`
- **状態**: ✅ **実装完了**
- **詳細**: Googleログイン、ログアウト、認証状態の監視など、設計された基本機能はすべて実装済みです。

### `usePets.ts`
- **状態**: 🟡 **一部実装**
- **詳細**: ペット情報を**読み取る**機能は実装されていますが、**書き込む（管理する）**機能が不足しています。
- **差分リスト**:
  - [ ] **ペット追加機能**: `addPet`関数は存在しますが、UIがないためユーザーはペットを追加できません。
  - [ ] **ペット編集機能**: 未実装です。
  - [ ] **ペット削除機能**: 未実装です。
  - [ ] **ペット選択機能**: `selectPet`関数は存在しますが、UIがないためユーザーは表示するペットを切り替えられません。

### `useLogbook.ts`
- **状態**: 🔴 **未実装に近い**
- **詳細**: ご指摘の通り、現在の実装は「リストの先頭にいるペットの、今日のログを読み書きする」という非常に限定的な機能しかありません。ペットを選択するUIがないため、「選択中のペット」という概念が機能しておらず、実質的に単一のペットのログしか扱えない状態です。設計書が意図するログ管理機能の多くが未実装です。
- **差分リスト**:
  - [ ] **過去ログの取得**: `where('timestamp', '>=', startOfDay)` という条件で**今日**のログに絞っているため、履歴画面で必要となる「指定した日付のログを取得するロジック」が未実装です。
  - [ ] **ログの編集機能**: 未実装です。
  - [ ] **ログの削除機能**: 未実装です。
  - [ ] **タスク管理連携**: タスクのCRUD（作成・更新・削除）に関するロジックは一切実装されていません。

## 3. UIレイヤー (Components & Pages)

### `page.tsx` (ホーム画面)
- **状態**: 🟡 **一部実装**
- **詳細**: ログ機能のコア部品は配置されていますが、設計書にある画面全体のレイアウトやナビゲーション、手動入力機能が欠けています。
- **差分リスト**:
  - [ ] **ヘッダー (`<Header />`)**: 未実装です。ペットの切り替えや設定画面への導線がありません。
  - [ ] **フッター (`<FooterNav />`)**: 未実装です。他の主要画面へのナビゲーションがありません。
  - [ ] **手動ログ追加機能**: フローティングボタン (`<QuickAddButton />`) と、そこから開く入力モーダル (`<ManualAddLogModal />`) が未実装です。

### `TaskSelector.tsx`
- **状態**: 🟡 **一部実装**
- **詳細**: ボタンの表示とログ追加の基本動作は実装済みですが、UI/UXが設計と異なります。
- **差分リスト**:
  - [ ] **タスクカラー**: `task.color`の値をボタンの背景色に反映する機能が未実装です（コード内に`TODO`コメントあり）。
  - [ ] **レイアウト**: 設計では「横スクロール」が示唆されていますが、現在は「折り返し (`flex-wrap`)」で実装されています。

### `LogTimeline.tsx`
- **状態**: 🟡 **一部実装**
- **詳細**: ログ表示の基本はできていますが、インタラクション（操作）と詳細情報の表示が不足しています。
- **差分リスト**:
  - [ ] **編集・削除ボタン**: 各ログアイテムに対する操作ボタンが未実装です。
  - [ ] **メモ表示**: ログにメモ（`note`）が紐づいている場合に、それを表示する機能が未実装です。

## 4. 総括
現状の実装は「**Googleログインして、今日の行動をワンタップで記録し、タイムラインで見る**」という、MVP（Minimum Viable Product）のコア部分に特化している状態です。

設計書にあるような「**ログやタスクの管理・編集機能**」や「**複数日にまたがるデータ操作**」は、これから開発が必要な段階です。
