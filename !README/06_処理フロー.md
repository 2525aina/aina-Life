# 06_処理フロー

このドキュメントは、`aina-Life` アプリケーションの主要な処理の流れと、各ファイルの役割を解説します。

## 1. 設計思想

このアプリケーションは、関心の分離を意識して構築されています。

-   **UIコンポーネント (`/components`)**: 見た目（UI）の表示に特化し、具体的なロジックはフックから受け取ります。
-   **カスタムフック (`/hooks`)**: Firebaseとの通信や状態管理などの複雑なロジックをカプセル化し、UIコンポーネントから簡単に利用できるようにします。
-   **Firebase設定 (`/lib`)**: Firebaseへの接続設定を単一のファイルに集約します。

これにより、各ファイルの見通しが良くなり、機能の追加や修正が容易になっています。

## 2. 主要ファイルの役割

### `src/lib/firebase.ts`
-   **役割**: **Firebaseへの接続**
-   **詳細**: アプリケーションとFirebaseバックエンド（認証、データベース）を接続するための初期設定を行います。APIキーなどの接続情報は環境変数から読み込まれ、`auth`（認証用）と `db`（データベース用）のインスタンスを生成し、他のファイルへ提供します。**すべてのFirebase連携の起点**となるファイルです。

### `src/app/layout.tsx`
-   **役割**: **アプリケーションの共通レイアウト**
-   **詳細**: 全ページで共通となるHTMLの骨格（`<html>`, `<body>`タグ）やフォントを定義します。

### `src/app/page.tsx`
-   **役割**: **メインページの制御**
-   **詳細**: アプリケーションのトップページです。`useAuth`フックを利用してユーザーのログイン状態を判定し、表示するコンポーネントを切り替える**司令塔**の役割を果たします。
    -   **未ログイン時**: `LoginButton` を表示します。
    -   **ログイン時**: `UserProfile`, `TaskSelector`, `LogTimeline` を表示します。

---

### `src/hooks/useAuth.ts`
-   **役割**: **認証ロジックのカプセル化**
-   **詳細**: Googleログイン、ログアウト、ユーザーの認証状態の監視といった、Firebase Authenticationに関するすべてのロジックを内包します。このフックのおかげで、UIコンポーネントは認証処理の複雑さを意識する必要がありません。

### `src/hooks/usePets.ts`
-   **役割**: **ペット情報の管理**
-   **詳細**: ログインユーザーに紐づくペットの一覧をFirestoreから取得します。また、現在どのペットが選択されているか (`selectedPetId`) を状態として管理し、アプリケーション全体で共有します。

### `src/hooks/useLogbook.ts`
-   **役割**: **タスクとログのデータ操作**
-   **詳細**: アプリケーションの**中核的なデータロジック**を担当します。`usePets`から受け取った `selectedPetId` に基づいて、そのペットのタスク一覧と行動ログをFirestoreからリアルタイムで取得します。また、新しいログをデータベースに書き込む `addLog` 関数を提供します。

---

### `src/components/LoginButton.tsx`
-   **役割**: **ログインボタンUI**
-   **詳細**: `useAuth`フックから `signInWithGoogle` 関数を受け取り、Googleログインボタンのクリックイベントと結びつけます。

### `src/components/UserProfile.tsx`
-   **役割**: **ユーザー情報とログアウトUI**
-   **詳細**: `useAuth`フックからユーザー情報と `signOutUser` 関数を受け取り、ユーザー名の表示とログアウト機能を提供します。

### `src/components/TaskSelector.tsx`
-   **役割**: **タスク選択ボタンUI**
-   **詳細**: `useLogbook`フックからタスク一覧を受け取ってボタンとして表示し、クリックされたら同フックの `addLog` 関数を呼び出して行動を記録します。

### `src/components/LogTimeline.tsx`
-   **役割**: **行動ログのタイムラインUI**
-   **詳細**: `useLogbook`フックからログの一覧を受け取り、時系列に整形して表示します。Firestoreのデータが更新されると、リアルタイムで表示に反映されます。

## 3. 処理フローの図解

### ① 起動〜ログインまで
```
ユーザー
  │
  v
[ page.tsx ] --(ログイン状態を確認)--> [ useAuth ]
  │
  ├─ (未ログイン) ─> [ LoginButton ] を表示
  │      │
  │      └─ (クリック) ─> [ useAuth.signInWithGoogle ] を実行
  │                         │
  │                         v
  │                      [ Firebase Auth ] で認証
  │                         │
  └<──(認証状態が変化)<───────┘
```

### ② ログイン後のデータ表示
```
[ useAuth ] --(ユーザー情報)--> [ usePets ]
  │                                │
  │                                v
  │                             [ Firestore ] からペット一覧を取得
  │                                │
  │                                v
  └──────────> [ useLogbook ] <──(選択中のペットID)<──┘
                   │
                   v
                [ Firestore ] からタスクとログを取得
                   │
                   v
[ page.tsx ] <─(タスクとログ)─┘
  │
  ├─> [ UserProfile ]
  ├─> [ TaskSelector ]
  └─> [ LogTimeline ]
```

### ③ 行動ログの記録
```
[ TaskSelector ] --(ボタンクリック)--> [ useLogbook.addLog ]
      │                                       │
      │                                       v
      │                                    [ Firestore ] に新しいログを書き込み
      │                                       │ (リアルタイム更新)
      └<──(更新されたログ一覧)<─[ LogTimeline ]<─┘
```
