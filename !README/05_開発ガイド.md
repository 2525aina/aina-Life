# 05_開発ガイド

このドキュメントは、犬ライフログアプリ「aina-Life」の開発に関する総合的なガイドラインです。プロジェクトの全体像、開発原則、現在の実装状況、今後の計画を素早く把握できます。

---

## 1. プロジェクト概要

aina-Lifeは、犬の1日の行動を簡単に記録・管理し、複数ユーザーでの共有を可能にするライフログアプリケーションです。ワンタップ記録、ログの閲覧・管理、複数ペット対応、Google認証などを主要機能としています。

---

## 2. 開発原則と規約

### 2.1 コーディング規約と設計思想
開発における品質と一貫性を保つためのルールです。

-   **ロジックの分離：カスタムフックの徹底活用**
    -   **指針**: 「ユーザー認証」や「データ取得」のようなビジネスロジックは、UIコンポーネントから完全に分離し、`src/hooks`ディレクトリに**カスタムフック**としてカプセル化します。
-   **コンポーネントの責務：自己完結型コンポーネント**
    -   **指針**: 各コンポーネントは、親から指示（Props）を待つだけでなく、**自身が必要とするデータや関数を、自身でフックを呼び出して取得**します。これにより、コンポーネントの再利用性が高まり、疎結合な設計になります。
-   **状態管理：React標準機能の活用**
    -   **指針**: グローバルな状態管理のために、すぐにReduxのような外部ライブラリを導入せず、まずはReact標準の`useState`, `useEffect`, `Context`（フック経由で利用）で実現することを基本方針とします。

### 2.2 技術スタック
本プロジェクトは以下の主要技術を使用しています。詳細については `02_構造.md` を参照してください。

-   **フレームワーク**: Next.js (App Router)
-   **言語**: TypeScript
-   **UIライブラリ**: React
-   **バックエンドサービス**: Firebase (Authentication, Firestore, Functions, Hosting)
-   **スタイリング**: Tailwind CSS

## 3. アプリケーション処理フロー

aina-Lifeアプリケーションの主要な処理の流れを概念的に説明します。各画面や機能の**詳細な設計**については `03_設計.md` を参照してください。

### 3.1. 設計思想
アプリケーションは関心の分離を意識して構築されており、UIコンポーネント、カスタムフック、Firebase設定がそれぞれの役割を担います。

### 3.2. 主要ファイルの役割
-   **`src/lib/firebase.ts`**: Firebaseへの接続とインスタンスのエクスポート。
-   **`src/app/layout.tsx`**: アプリケーションの共通レイアウト。
-   **`src/app/page.tsx`**: メインページの制御（認証状態に応じたUI切り替え）。
-   **`src/hooks/useAuth.ts`**: 認証ロジックのカプセル化。
-   **`src/hooks/usePets.ts`**: ペット情報の管理。
-   **`src/hooks/useLogbook.ts`**: タスクとログのデータ操作。
-   **`src/components/LoginButton.tsx`**: ログインボタンUI。
-   **`src/components/UserProfile.tsx`**: ユーザー情報とログアウトUI。
-   **`src/components/TaskSelector.tsx`**: タスク選択ボタンUI。
-   **`src/components/LogTimeline.tsx`**: 行動ログのタイムラインUI。

### 3.3. 処理フローの図解

#### ① 起動〜ログインまで
```
ユーザー
  │
  v
[ page.tsx ] --(ログイン状態を確認)--> [ useAuth ]
  │
  ├─ (未ログイン) ─> [ LoginButton ] を表示
  │      │
  │      └─ (クリック) ─> [ useAuth.signInWithGoogle ] を実行
  │                         │
  │                         v
  │                      [ Firebase Auth ] で認証
  │                         │
  └<──(認証状態が変化)<───────┘
```

#### ② ログイン後のデータ表示
```
[ useAuth ] --(ユーザー情報)--> [ usePets ]
  │                                │
  │                                v
  │                             [ Firestore ] からペット一覧を取得
  │                                │
  │                                v
  └──────────> [ useLogbook ] <──(選択中のペットID)<──┘
                   │
                   v
                [ Firestore ] からタスクとログを取得
                   │
                   v
[ page.tsx ] <─(タスクとログ)─┘
  │
  ├─> [ UserProfile ]
  ├─> [ TaskSelector ]
  └─> [ LogTimeline ]
```

#### ③ 行動ログの記録
```
[ TaskSelector ] --(ボタンクリック)--> [ useLogbook.addLog ]
      │                                       │
      │                                       v
      │                                    [ Firestore ] に新しいログを書き込み
      │                                       │ (リアルタイム更新)
      └<──(更新されたログ一覧)<─[ LogTimeline ]<─┘
```

## 4. 実装状況と今後の計画

### 4.1 実装済み機能の概要
これまでに以下の主要機能が実装されています。

-   **Googleログイン機能**: Firebase Authentication を用いたGoogleアカウントでの認証・ログアウト機能。
-   **ペット選択機能と基本レイアウト**: ヘッダーにペット選択UIを組み込み、全画面共通のヘッダーとフッターを配置。
-   **ワンタップ記録機能**: タスクボタンをタップすることで、現在時刻とタスク名をFirestoreに自動記録する機能。
-   **手動ログ記録機能**: 時刻やメモを指定してログを記録する機能。
-   **ログタイムライン機能**: 記録されたログをホーム画面にリアルタイムで表示する機能。
-   **履歴画面**: 日付を切り替えて過去のログを閲覧・編集・削除する機能。**ログ編集モーダルの安定性が向上しました。**
-   **タスク管理画面**: タスクのCRUD（追加・編集・削除）および並べ替え機能。

### 4.2 設計と実装の差分
現在の設計書と実装状況の間の主要な差分です。

-   **`usePets.ts`**: ペット情報の**読み取り**と**選択**機能は実装済みであり、**追加・編集・削除**機能もフックレベルで実装済みです。UIはまだ不足しています。
-   **`TaskSelector.tsx`**: タスクに設定した色をボタンの背景色に反映する機能が未実装です。
-   **ペット管理画面 (`/pets`)**: 画面全体が未実装です。
-   **家族と共有機能**: 未実装です。
-   **分析・統計機能**: 未実装です。

### 4.3 今後の実装計画
以下の順序で開発を進めるのが、最も効率的で手戻りの少ないベストプラクティスです。

#### ステップ1：ペット管理機能のUI実装
1.  **実装内容**: `/pets` ルーティング設定、ペットの追加・編集・削除を行うUIを実装します。**フックレベルでの機能は既に実装済みです。**
2.  **理由**: アプリケーションの中核となるペット情報をユーザーが管理できるようにするため。

#### ステップ2：タスクボタンのカラー反映
1.  **実装内容**: ホーム画面のタスクボタンに、ユーザーが設定した色を反映させます。
2.  **理由**: UIの視認性とカスタマイズ性を向上させるため。

#### ステップ3：家族と共有機能
1.  **実装内容**: 1つのペットの情報を複数ユーザーで共有するためのUIとロジックを実装します。
2.  **理由**: アプリの主要要件の一つであり、複数人での利用を可能にするため。

#### ステップ4：統計・分析機能
1.  **実装内容**: 記録されたログデータを集計し、グラフなどで可視化する画面を実装します。
2.  **理由**: アプリの付加価値を高め、ユーザーにインサイトを提供するため。
