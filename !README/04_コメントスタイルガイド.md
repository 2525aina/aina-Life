# コメントスタイルガイド（学習用・丁寧型）

## 目的
このガイドは、**学習者や新規参加メンバーでも理解しやすいコメントの書き方**を統一するための基準です。コードの読みやすさと保守性を高めることを目的とします。  

---

## 1. ファイル冒頭
- ファイルの責務を **1〜3行で記載**  
- 依存する主要なフック・コンポーネントも合わせて明記  

✅ 良い例
``` ts
// src/components/UserProfile.tsx
// ユーザー情報を表示するコンポーネント。
// 依存: useAuth, Avatar, ProfileCard
```

❌ 悪い例
``` ts
// UserProfile コンポーネントです
```

---

## 2. import 部分
- **用途が直感で分かるもの** → コメント不要  
- **用途が不明瞭なもの** → 1行で補足  

✅ 良い例
``` ts
import { QuickAddButton } from "@/components/QuickAddButton"; // ログ追加用のフローティングボタン
```

---

## 3. 変数・フック
- 「何をしているか」ではなく **「なぜ必要か」** を説明  
- 特に `useState`, `useEffect`, useContext などは意図を残す  

✅ 良い例
``` ts
const [isModalOpen, setIsModalOpen] = useState(false); // 手動ログ追加モーダルの開閉状態を管理
```

❌ 悪い例
``` ts
const [isModalOpen, setIsModalOpen] = useState(false); // useStateでisModalOpenを作成
```

---

## 4. 条件分岐
- UIの分岐には **ユーザー体験の意図** を記載  

✅ 良い例
``` ts
if (loading) {
  // 認証状態が確定するまでローディング表示
}
```

---

## 5. JSX 内
- **props の意味が直感的でない場合** にコメントを残す  
- コンポーネント配置の意図を1行で説明  

✅ 良い例
``` tsx
<LogTimeline logs={logs} /> {/* ユーザーの行動履歴を時系列で表示 */}
```

---

## 6. コメントの文体ルール
- **「です・ます」調**で統一  
- 1行は **40文字程度以内**  
- **コードと重複しない表現**にする  
- 「どうやって」ではなく **「なぜ」** を優先  
- 必要に応じて **英語と日本語を混ぜても良い**（例: useAuth → 認証フック）  

---

## 7. 書かない方がいいコメント
- **コードを直訳しただけ**  
``` ts
const { user } = useAuth(); // useAuthからuserを取得 ← 不要
```
- **常識的な処理**  
``` ts
return <div>...</div>; // divを返す ← 不要
```

---

## 8. 例外ルール
- **難解な処理や仕様依存部分** → 詳細に補足してよい  
- **暫定対応や技術的負債** → // TODO: や // FIXME: を明示  

✅ 良い例
``` ts
// TODO: APIが正式リリースされたら暫定エンドポイントを修正する
```

---

## まとめ
- **ファイル単位で責務を明確に書く**  
- **コードの「意図」を伝えるコメントにする**  
- **冗長な逐語訳コメントは書かない**  
- **例外や将来の修正点はTODOで明記**  

---

## 参考例

```ts
// src/lib/firebase.ts
// Firebaseの初期化および各サービスインスタンスの取得。
// 依存: 環境変数(NEXT_PUBLIC_FIREBASE_*)、firebase/app, firebase/auth, firebase/firestore

import { initializeApp, getApps, getApp } from 'firebase/app'; // Firebaseアプリ初期化
import { getAuth } from 'firebase/auth'; // 認証サービス
import { getFirestore } from 'firebase/firestore'; // Firestoreデータベース

// Firebaseの設定は環境変数から読み込む
const firebaseConfig = {
  apiKey: process.env.NEXT_PUBLIC_FIREBASE_API_KEY, // APIキー
  authDomain: process.env.NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN, // 認証用ドメイン
  projectId: process.env.NEXT_PUBLIC_FIREBASE_PROJECT_ID, // プロジェクトID
  storageBucket: process.env.NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET, // ストレージバケット
  messagingSenderId: process.env.NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID, // メッセージ送信者ID
  appId: process.env.NEXT_PUBLIC_FIREBASE_APP_ID, // アプリID
  measurementId: process.env.NEXT_PUBLIC_FIREBASE_MEASUREMENT_ID, // アナリティクス用ID
};

// Firebaseアプリの初期化
// 既に初期化済みの場合は既存のアプリを再利用して多重初期化を防ぐ
const app = !getApps().length ? initializeApp(firebaseConfig) : getApp();

// Firebase Authenticationのインスタンスを取得
const auth = getAuth(app);

// Firestoreデータベースのインスタンスを取得
const db = getFirestore(app);

// 他モジュールで利用できるようエクスポート
export { app, auth, db };
```
