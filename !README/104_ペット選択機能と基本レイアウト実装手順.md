# 104_ペット選択機能と基本レイアウト実装手順

このセクションでは、アプリケーションの基本レイアウト（ヘッダーとフッター）と、ヘッダーに組み込まれたペット選択機能の実装手順を説明します。これにより、アプリの骨格が確立され、複数ペット対応の基盤が整います。

---

## 1. ヘッダーコンポーネント (`Header.tsx`) の作成と実装

### 1.1. コンポーネントファイルの作成

`src/components/Header.tsx` を作成します。

<details>
<summary>クリックしてコードを表示</summary>

```typescript
// src/components/Header.tsx
'use client';

import React from 'react';
import { usePets } from '@/hooks/usePets';
import Link from 'next/link'; // 設定アイコンのリンク用

export const Header = () => {
  const { pets, selectedPetId, selectPet, loading } = usePets();

  const handlePetChange = (e: React.ChangeEvent<HTMLSelectElement>) => {
    selectPet(e.target.value);
  };

  // ローディング中でもUIのガタつきを抑えるためのスケルトン
  if (loading) {
    return (
      <header className="w-full p-4 bg-white border-b border-gray-200">
        <div className="max-w-lg mx-auto flex justify-between items-center">
          <div className="h-7 w-24 bg-gray-200 rounded animate-pulse"></div>
          <div className="h-6 w-6 bg-gray-200 rounded-full animate-pulse"></div>
        </div>
      </header>
    );
  }

  return (
    <header className="w-full p-4 bg-white border-b border-gray-200 sticky top-0 z-10">
      <div className="max-w-lg mx-auto flex justify-between items-center">
        {pets.length > 0 && selectedPetId ? (
          <select
            value={selectedPetId}
            onChange={handlePetChange}
            className="text-lg font-bold bg-transparent border-none focus:ring-0 p-0 text-gray-800"
          >
            {pets.map((pet) => (
              <option key={pet.id} value={pet.id}>
                {pet.name}
              </option>
            ))}
          </select>
        ) : (
          <div className="text-lg font-bold text-gray-800">ペットを登録してください</div>
        )}
        {/* TODO: 設定アイコンのリンク先を後ほど実装 */}
        <Link href="/settings" className="text-gray-600 hover:text-gray-800">
          ⚙️
        </Link>
      </div>
    </header>
  );
};
```

</details>

### 1.2. 実装のポイント

-   `usePets`フックから `pets` (ペット一覧)、`selectedPetId` (選択中のペットID)、`selectPet` (ペット選択関数)、`loading` (ローディング状態) を取得します。
-   `selectedPetId` を `<select>` 要素の `value` にバインドし、`onChange` イベントで `selectPet` を呼び出すことで、ペットの切り替えを実現します。
-   `loading` 状態に応じてスケルトンUIを表示し、ユーザー体験を向上させます。
-   ペットが登録されていない場合は、「ペットを登録してください」というメッセージを表示します。
-   設定アイコンは仮で `⚙️` を表示し、`/settings` へのリンクを設定しています（リンク先は後で実装）。

## 3. メインページ (`page.tsx`) への統合

作成したヘッダーとフッターを `src/app/page.tsx` に組み込み、アプリケーションの基本レイアウトを完成させます。

### 3.1. `page.tsx` の修正

<details>
<summary>クリックしてコードを表示</summary>

```typescript
// src/app/page.tsx
// このファイルはアプリケーションのメインページであり、認証状態に基づいてUIを切り替えます。
// 認証ロジックはカスタムフックに、UIコンポーネントは個別のファイルに分割されています。

"use client"; // このコンポーネントはクライアントサイドで実行されます。

import { useAuth } from "@/hooks/useAuth"; // 作成した認証カスタムフックをインポート
import LoginButton from "@/components/LoginButton"; // ログインボタンコンポーネントをインポート
import UserProfile from "@/components/UserProfile"; // ユーザープロフィールコンポーネントをインポート
import { TaskSelector } from "@/components/TaskSelector"; // タスク選択コンポーネント
import { LogTimeline } from "@/components/LogTimeline"; // ★追加：ログタイムラインコンポーネント
import { Header } from "@/components/Header"; // 追加
import { FooterNav } from "@/components/FooterNav"; // 追加

// Homeコンポーネントの定義
export default function Home() {
  // useAuthフックから認証ユーザー情報とローディング状態を取得します。
  const { user, loading } = useAuth();

  // 認証状態の読み込み中の場合、ローディングメッセージを表示します。
  if (loading) {
    return (
      <div className="min-h-screen flex items-center justify-center">
        <p>読み込み中...</p>
      </div>
    );
  }

  return (
    <div className="flex flex-col min-h-screen bg-gray-50"> {/* 背景色を修正 */}
      <Header />
      <main className="flex-grow w-full pb-16"> {/* フッターの高さ分パディングを追加 */}
        {user ? (
          <div className="max-w-lg mx-auto p-4 space-y-4">
            <UserProfile />
            <TaskSelector />
            <LogTimeline />
          </div>
        ) : (
          <div className="min-h-full flex flex-col items-center justify-center">
            <h1 className="text-3xl font-bold mb-8">aina Life アプリ</h1>
            <LoginButton />
          </div>
        )}
      </main>
      <FooterNav />
    </div>
  );
}
```

</details>

### 3.2. 実装のポイント

-   `Header` と `FooterNav` をインポートし、メインのレイアウトに配置します。
-   `main` タグに `pb-16` (padding-bottom: 4rem) を追加し、フッターがコンテンツに重ならないように調整します。
-   全体の背景色を `bg-gray-50` に変更し、より自然な見た目にします。
