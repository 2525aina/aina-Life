# aina-Life 実装手順書 - ペット管理機能

## 1. 概要

このセクションでは、ハードコードされたペットID (`TEMP_DOG_ID`) を廃止し、ユーザーが自身のペット情報を管理し、記録対象のペットを切り替えられるようにする機能の実装手順を計画します。

これにより、設計書に定められた「複数ペット対応」の基盤を構築します。

---

## 2. 実装計画

### 2.1. ペット管理用カスタムフックの作成 (`usePets.ts`)

まず、ペット情報の取得と状態管理ロジックをカプセル化した、新しいカスタムフックを`src/hooks`に作成します。

- **ファイルパス**: `src/hooks/usePets.ts`
- **状態 (State)**:
  - `pets`: ログインユーザーがオーナーであるペットのリスト (`Pet[]`)
  - `selectedPetId`: 現在選択されているペットのID (`string | null`)
  - `loading`: データ読み込み中の状態 (`boolean`)
- **機能**:
  - `useEffect`内で、Firestoreの`dogs`コレクションから`ownerIds`配列に現在のユーザーIDが含まれるドキュメントをクエリし、`pets`ステートを更新する。
  - `addPet(petData)`: 新しいペットを`dogs`コレクションに追加する関数。
  - `selectPet(petId)`: `selectedPetId`ステートを更新する関数。
- **返り値**: `{ pets, selectedPetId, loading, addPet, selectPet }`

### 2.2. `useLogbook`フックの修正

既存の`useLogbook`フックを、新しく作る`usePets`フックと連携するように修正します。

- **ファイルパス**: `src/hooks/useLogbook.ts`
- **修正内容**:
  - フックの先頭で`usePets()`を呼び出し、`selectedPetId`を取得します。
  - `TEMP_DOG_ID`を使用しているすべての箇所を、`selectedPetId`に置き換えます。
  - `useEffect`の依存配列に`selectedPetId`を追加し、選択中のペットが切り替わった際に、タスクとログの再取得が実行されるようにします。
  - `selectedPetId`が存在しない場合は、クエリを実行しないように制御します。

### 2.3. ペット選択UIコンポーネントの作成

ユーザーがペットを選択するためのUIコンポーネントを`src/components`に作成します。

- **ファイルパス**: `src/components/PetSelector.tsx`
- **機能**:
  - `usePets()`を呼び出し、ペットのリスト`pets`、選択中のID `selectedPetId`、および選択関数`selectPet`を取得します。
  - ペットのリストを元に、ドロップダウンメニュー(`<select>`)またはボタンのリストをレンダリングします。
  - ユーザーがUIを操作してペットを選択した際に、`selectPet(petId)`を呼び出します。

### 2.4. メインページへの統合

作成した`<PetSelector />`をメインページに組み込みます。

- **ファイルパス**: `src/app/page.tsx`
- **修正内容**: ヘッダーやプロファイル表示エリアなど、適切な場所に`<PetSelector />`コンポーネントを配置します。

### 2.5. ペット追加機能の実装（次のステップ）

ペット管理画面 (`/pets`) を新しく作成し、そこで`usePets`フックの`addPet`関数を呼び出すフォームを実装します。これは、ペット選択機能が完成した後のステップとします。

## 3. 完了の定義

- [ ] アプリケーションからハードコードされたペットID (`TEMP_DOG_ID`) が完全に排除される。
- [ ] ログイン後、ユーザーは自分が登録したペットの一覧から、ログを記録したい対象を選択できる。
- [ ] ペットを選択すると、そのペットに紐づくタスクボタンとログタイムラインが正しく表示される。
- [ ] ペットが1匹も登録されていない場合、その旨をユーザーに伝えるメッセージが表示される。
