# 14_ペット管理機能実装手順

このドキュメントは、「aina-Life」アプリにペット管理機能（追加・編集・削除）を実装するための詳細な手順を記述します。

設計書 `03_設計.md` およびコーディング規約 `05_コーディング規約.md` に準拠します。

---

## 目的

ユーザーが所有するペットの情報を、アプリケーション内で完全に管理できるようにする。

-   新しいペットを登録できる。
-   既存のペット情報を編集できる。
-   登録したペットを削除できる。

---

## 実装ステップ

### ステップ1: `usePets`フックの機能拡張

まず、データ操作のロジックを担う `src/hooks/usePets.ts` に、ペットの編集・削除機能を追加します。

1.  **`updatePet` 関数の実装**:
    -   引数: `petId: string`, `petData: Partial<Pet>`
    -   処理: Firestoreの `doc(db, 'dogs', petId)` で対象ドキュメントへの参照を取得し、`updateDoc` を使って `petData` の内容で更新する。

2.  **`deletePet` 関数の実装**:
    -   引数: `petId: string`
    -   処理: `doc(db, 'dogs', petId)` で参照を取得し、`deleteDoc` を使ってドキュメントを削除する。
    -   **注意**: 関連するサブコレクション（`tasks`, `logs`）の削除も考慮する必要がありますが、初期実装ではペット本体の削除のみとします。（将来的にCloud Functionでの一括削除を検討）

3.  **`addPet` 関数の確認**:
    -   既存の `addPet` 関数が、設計通りのデータ（名前、犬種、誕生日など）を受け取れることを確認します。

### ステップ2: UIコンポーネントの作成

次に、ユーザーが操作するための画面と部品を作成します。

1.  **ルーティング設定**:
    -   `src/app/pets/page.tsx` を作成し、ペット管理画面のルートを `/pets` に設定します。

2.  **ペット管理ページ (`PetManagementPage`)**:
    -   `src/app/pets/page.tsx` に、画面のメインコンテナとなるコンポーネントを作成します。
    -   このコンポーネント内で `usePets` フックを呼び出し、ペットの一覧を取得します。

3.  **ペット一覧コンポーネント (`PetList`)**:
    -   `src/components/PetList.tsx` を作成します。
    -   `usePets` から取得した `pets` 配列を元に、ペットの一覧を表示します。
    -   各ペット項目には「編集」ボタンと「削除」ボタンを配置します。
    -   リストの上部または下部に「新しいペットを追加」ボタンを配置します。

4.  **ペット編集/追加モーダル (`PetEditModal`)**:
    -   `src/components/PetEditModal.tsx` を作成します。
    -   新規追加と既存編集の両方で使えるように、汎用的なフォームとして実装します。
    -   フォームには、名前・犬種・誕生日の入力フィールドと「保存」ボタンを設置します。
    -   「保存」ボタンクリックで、`addPet` または `updatePet` を呼び出します。

5.  **削除確認モーダル (`DeleteConfirmationModal`)**:
    -   `src/components/DeleteConfirmationModal.tsx` を作成します。
    -   誤操作を防ぐため、「本当に削除しますか？」という確認メッセージと、「はい」「いいえ」ボタンを表示します。
    -   「はい」をクリックすると `deletePet` を呼び出します。

### ステップ3: UIとロジックの接続

最後に、作成したUIコンポーネントとフックのロジックを結びつけます。

1.  **`PetManagementPage`**:
    -   「新しいペットを追加」ボタンがクリックされたら、`PetEditModal` を「追加モード」で開きます。
    -   `PetList` 内の「編集」ボタンがクリックされたら、対象のペット情報を渡し、`PetEditModal` を「編集モード」で開きます。
    -   `PetList` 内の「削除」ボタンがクリックされたら、`DeleteConfirmationModal` を開きます。

2.  **`PetEditModal`**:
    -   フォームの「保存」ボタンが押された際、モード（追加/編集）に応じて `usePets` の `addPet` または `updatePet` を呼び出します。
    -   処理完了後はモーダルを閉じ、ペット一覧を再表示させます。

3.  **`DeleteConfirmationModal`**:
    -   「はい」ボタンが押されたら、`usePets` の `deletePet` を呼び出し、モーダルを閉じます。

---

この手順に沿って実装することで、設計書通りのペット管理機能が完成します。
