# aina-Life 開発環境セットアップ手順書

## 1. 概要
このドキュメントは、`aina-Life` アプリケーションの開発環境をローカルマシンに構築するための手順を記述します。

---

## 2. 前提条件
開発を始める前に、お使いのコンピュータに以下のソフトウェアがインストールされていることを確認してください。

*   **Node.js**: `v18.17` 以上
*   **npm**: `v9.6.7` 以上 (通常はNode.jsに同梱されています)

ターミナルで以下のコマンドを実行して、バージョンを確認できます。
```bash
node -v
npm -v
```

---

## 3. プロジェクトのセットアップ

### A. GitHubからプロジェクトをクローンした場合

1.  以下のコマンドを実行して、プロジェクトをクローンします。
    ```bash
    git clone https://github.com/2525aina/aina-Life.git
    ```
2.  プロジェクトディレクトリに移動します。
    ```bash
    cd aina-Life
    ```
3.  依存関係をインストールします。
    ```bash
    npm install
    ```
4.  次の「4. Firebase プロジェクト連携」に進んでください。

---

### B. 新規にプロジェクトを作成する場合

1.  任意の作業ディレクトリで、ターミナルから以下のコマンドを実行します。
    このコマンドは、`aina-Life`という名称の新しいフォルダを作成し、その中にアプリケーションの雛形をセットアップします。

    ```bash
    npx create-next-app@latest aina-Life --typescript --eslint --tailwind --src-dir --app --import-alias "@/*" --use-npm --no-turbopack
    ```

    **コマンドオプションの内訳:**
    *   `--typescript`: TypeScriptを導入します。
    *   `--eslint`: コードの静的解析ツールESLintを導入します。
    *   `--tailwind`: CSSフレームワークのTailwind CSSを導入します。
    *   `--src-dir`: ソースコードを `src` ディレクトリ内に整理します。
    *   `--app`: 最新のApp Routerを使用します。
    *   `--import-alias "@/*"`: `@/components/` のような絶対パスでのインポートを有効にします。

2.  作成されたプロジェクトディレクトリに移動します。
    ```bash
    cd aina-Life
    ```
3.  Firebaseをプロジェクトで利用するために、Firebase SDK (Web) をインストールします。
    ```bash
    npm install firebase
    ```
4.  次の「4. Firebase プロジェクト連携」に進んでください。


---

## 4. Firebase プロジェクト連携

1.  **Firebase プロジェクトの作成**
    *   [Firebaseコンソール](https://console.firebase.google.com/)にアクセスし、Googleアカウントでログインします。
    *   「プロジェクトを追加」をクリックし、画面の指示に従って新しいFirebaseプロジェクトを作成します。(プロジェクト名は `aina-Life-dev` などが推奨されます)

2.  **Webアプリの登録と設定キーの取得**
    *   プロジェクトの概要ページで、ウェブアイコン (`</>`) をクリックして新しいウェブアプリを登録します。
    *   アプリのニックネーム（例: `aina-Life-web`）を登録し、「アプリを登録」をクリックします。
    *   「Firebase SDK を追加」のステップで表示される `firebaseConfig` オブジェクトをコピーします。これは後で使います。

3.  **環境変数ファイルの設定**
    *   Next.jsプロジェクトのルート（`aina-Life`ディレクトリ直下）に、`.env.local` という名前のファイルを新規作成します。
    *   `.env.local` ファイルに、先ほどコピーした `firebaseConfig` の内容を以下のように貼り付け、キーの先頭に `NEXT_PUBLIC_` を付けます。

    ```
    NEXT_PUBLIC_FIREBASE_API_KEY="..."
    NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN="..."
    NEXT_PUBLIC_FIREBASE_PROJECT_ID="..."
    NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET="..."
    NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID="..."
    NEXT_PUBLIC_FIREBASE_APP_ID="..."
    ```
    **注意:** `.env.local` ファイルは `.gitignore` に含まれているため、APIキーなどの機密情報がGitリポジトリにアップロードされることはありません。

---

## 5. 開発サーバーの起動

1.  以下のコマンドを実行して、開発用のローカルサーバーを起動します。

    ```bash
    npm run dev
    ```

2.  ターミナルに `Ready on http://localhost:3000` と表示されたら、ブラウザで [http://localhost:3000](http://localhost:3000) にアクセスします。

3.  Next.jsのウェルカムページが表示されれば、環境構築は完了です。

---

## 6. (任意) Firebase Hostingでのデプロイ
開発中のアプリケーションをインターネット上に公開し、関係者と共有するための手順です。

### 6.1. Next.jsプロジェクトの設定変更
Firebase Hostingにデプロイするために、Next.jsが静的なHTML/CSS/JSファイルを生成するように設定を変更します。

1.  プロジェクトのルートにある `next.config.ts` ファイルを開きます。
2.  `output: 'export'` を追記して、ファイルを保存します。

    **変更前:**
    ```ts
    import type { NextConfig } from "next";

    const nextConfig: NextConfig = {
      /* config options here */
    };

    export default nextConfig;
    ```

    **変更後:**
    ```ts
    import type { NextConfig } from "next";

    const nextConfig: NextConfig = {
      output: 'export', // この行を追記
    };

    export default nextConfig;
    ```
    **注意:** この設定は、静的サイトとしてデプロイする場合に必要です。

### 6.2. Firebase CLIのセットアップ

1.  **Firebase Toolsのインストール**
    ターミナルで以下のコマンドを実行し、Firebaseのコマンドラインツールをグローバルにインストールします。
    ```bash
    npm install -g firebase-tools
    ```

2.  **Firebaseへのログイン**
    以下のコマンドを実行し、ブラウザでGoogleアカウントにログインしてFirebaseを認証します。
    ```bash
    firebase login
    ```

### 6.3. Hostingの初期設定

1.  プロジェクトのルートディレクトリ (`aina-Life`) で、以下のコマンドを実行します。
    ```bash
    firebase init hosting
    ```

2.  コマンド実行後、いくつか質問されますので、以下のように回答します。
    *   **Which Firebase project do you want to use?**
        *   矢印キーで `Use an existing project` → `aina-Life-dev` を選択し、Enterキーを押します。
    *   **What do you want to use as your public directory?**
        *   `out` と入力し、Enterキーを押します。（Next.jsが静的ファイルを生成するディレクトリです）
    *   **Configure as a single-page app (rewrite all urls to /index.html)?**
        *   `y` と入力し、Enterキーを押します。
    *   **Set up automatic builds and deploys with GitHub?**
        *   `n` と入力し、Enterキーを押します。

    これにより、`.firebaserc` と `firebase.json` の2つの設定ファイルが作成されます。

### 6.4. デプロイの実行

1.  **アプリケーションのビルド**
    以下のコマンドを実行して、Next.jsアプリケーションを静的ファイルとしてビルドします。
    ```bash
    npm run build
    ```
    これにより、`out` ディレクトリにデプロイ用のファイルが生成されます。

2.  **Firebaseへのデプロイ**
    最後に、以下のコマンドでビルドしたファイルをFirebase Hostingにデプロイします。
    ```bash
    firebase deploy
    ```

3.  デプロイが完了すると、ターミナルに `Hosting URL` が表示されます。このURLにアクセスすると、公開されたアプリケーションを確認できます。

## 7. 開発フローとGit運用

開発中に発生した問題の解決や、変更内容をリモートリポジトリに反映するまでの基本的な流れです。

### 7.1. 開発サーバー起動とエラー確認

まず、開発サーバーを起動して動作を確認します。

```bash
npm run dev
```

もし `Module not found` のようなエラーが出た場合、必要なパッケージが不足している可能性があります。

### 7.2. 不足パッケージのインストール

エラーメッセージで指定されたパッケージをインストールします。

```bash
npm install [パッケージ名]
# 例: npm install react-hot-toast
```

### 7.3. 変更内容の確認

パッケージのインストールやコードの修正が終わったら、`git status` で変更されたファイルを確認します。

```bash
git status
```

次に `git diff` で、具体的な変更内容を確認します。

```bash
git diff
```

### 7.4. コミットとプッシュ

1.  変更内容に問題がなければ、`git add` でステージングします。

    ```bash
    git add .
    ```

2.  `git commit` で変更をコミットします。メッセージは分かりやすく簡潔に記述します。

    ```bash
    git commit -m "feat: 機能追加や修正内容の要約"
    ```

    **主なコミット種別:**
    *   `feat`: 新機能の追加
    *   `fix`: バグ修正
    *   `docs`: ドキュメントの変更
    *   `style`: コードフォーマットの変更（コードの動作に影響しないもの）
    *   `refactor`: リファクタリング（機能追加やバグ修正を含まないコードの変更）
    *   `test`: テストの追加・修正
    *   `chore`: ビルドプロセスや補助ツールの変更など

3.  最後に `git push` でリモートリポジトリに変更を反映します。

    ```bash
    git push origin main
    ```
